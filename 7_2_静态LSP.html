静态LSP
    静态LSP简介
        静态LSP不能使用标签发布协议动态建立，需要由管理员手工配置，适用于拓扑结构简单并且稳定的网络
    ----------------------------------------------------------------------------------------------------------------------------------------------------------------------
    配置静态LSP
        前提：lsr之间路由可达
        1.配置LSR ID（系统视图下）需使用LSR某个Loopback接口的地址作为LSR ID
                mpls lsr-id lsr-id
        2.使能MPLS（系统视图下、需要转发MPLS报文的端口试图下）
                mpls
        3.建立静态LSP（系统视图下）需要遵循的原则是：前一节点出标签的值等于下一个节点入标签的值
            在Ingress进行
                static-lsp ingress lsp-name destination ip-address { mask-length | mask } nexthop next-hop-address out-label out-label
                例如：static-lsp ingress LSP1 destination 10.10.1.3 32 nexthop 10.1.1.2 out-label 20 
            在Transit进行
                static-lsp transit lsp-name incoming-interface interface-type interface-number in-label in-label  nexthop next-hop-address out-label out-label
                例如：static-lsp transit LSP1 incoming-interface gigabitethernet 1/0/0 in-label 20 nexthop 10.2.1.2 out-label 40 
            在Egress进行
                static-lsp egress lsp-name incoming-interface interface-type interface-number in-label in-label 
                例如：static-lsp egress LSP1 incoming-interface gigabitethernet 1/0/0 in-label 40 
    ----------------------------------------------------------------------------------------------------------------------------------------------------------------------
    查看静态LSP的信息
        display mpls static-lsp [ lsp-name ]
    ----------------------------------------------------------------------------------------------------------------------------------------------------------------------
    检测LSP的连通性
        ping lsp ip destination-address mask-length
        tracert lsp ip destination-address mask-length
